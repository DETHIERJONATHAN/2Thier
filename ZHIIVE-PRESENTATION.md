## 1. Résumé exécutif

Zhiive est une plateforme hybride qui réunit dans un même écosystème:

- activité sociale
- messagerie temps réel
- géolocalisation et navigation
- événements et participation
- visibilité locale
- transparence client
- outils métiers pour les sociétés
- automatisation issue de l'activité réelle

L'ambition n'est pas de créer un simple CRM, un simple portail client ou un simple réseau social, mais une plateforme vivante où utilisateurs et sociétés interagissent dans le même espace.

---

## 2. Vision

### Vision courte

Créer une ruche vivante où les utilisateurs et les sociétés coexistent, communiquent, se découvrent, travaillent, se rendent visibles et créent de la valeur dans un même environnement.

### Vision longue

Zhiive part d'une idée simple:

> le digital ne devrait plus séparer la vie sociale, la communication, la visibilité, la relation client, l'activité terrain et les outils opérationnels.

Dans Zhiive:

- les utilisateurs vivent une expérience sociale utile, locale et récurrente
- les sociétés existent comme des acteurs visibles, joignables et crédibles
- l'activité réelle peut devenir du contenu, de la confiance et de la transparence
- les clients peuvent suivre leur parcours directement dans la plateforme

---

## 3. Positionnement

### Ce que Zhiive n'est pas

Zhiive n'est pas:

- un simple CRM
- un simple ERP
- un simple portail client
- un simple réseau social
- un simple outil de chantier
- un simple système de navigation

### Ce qu'est Zhiive

Zhiive est:

- une plateforme sociale vivante
- une interface de proximité
- une carte active
- un système de communication unifié
- un espace de visibilité pour les sociétés
- un moteur de transformation activité -> contenu -> confiance -> opportunité

---

## 4. Publics cibles

### Utilisateurs

Les utilisateurs viennent dans Zhiive pour:

- interagir avec d'autres personnes
- suivre l'activité autour d'eux
- utiliser la carte et la proximité
- échanger via messagerie
- participer à des événements ou tournois
- découvrir des sociétés et des communautés

### Sociétés

Les sociétés utilisent Zhiive pour:

- exister dans un environnement vivant
- être vues, suivies et contactées
- montrer leur activité sans dépendre d'un marketing permanent
- piloter leads, devis, documents, chantiers, calendrier, facturation et relation client
- transformer leur activité réelle en crédibilité et opportunités

---

## 5. Stratégie go-to-market

Le principe central est simple:

> les utilisateurs attirent les sociétés.

La séquence de croissance visée est:

1. créer une forte valeur utilisateur
2. faire vivre l'écosystème
3. générer de la densité sociale et de l'usage récurrent
4. rendre la présence des sociétés naturellement attractive
5. convertir cette présence en opportunité business

Zhiive doit donc d'abord être compris comme une plateforme vivante et utile, pas comme un outil pro classique.

## 11. Ancrage dans le dépôt actuel

Le dépôt montre déjà une base technique forte pour cette vision:

- frontend React 19 + TypeScript + Vite 6
- backend Express 5
- Prisma + PostgreSQL
- PWA via `vite-plugin-pwa`
- tests Vitest
- CI/CD GitHub Actions -> Cloud Build -> Cloud Run
- serveur unifié qui sert API et frontend

On y voit également plusieurs domaines déjà présents:

- social
- messenger
- Wax
- Arena
- websites
- IA
- documents
- invoices
- expenses
- calendar
- Peppol
- rôles / permissions
- préférences utilisateur

Autrement dit: la vision produit n'est pas purement conceptuelle. Elle s'appuie sur une base déjà réelle.

## 12. Architecture technique

### Frontend

Le frontend repose sur:

- React
- Vite
- alias structurés
- code splitting
- compression gzip / brotli
- PWA
- manifest dynamique
- proxy API local

### Backend

Le backend repose sur un serveur Express unifié qui gère:

- sécurité
- sessions PostgreSQL
- upload
- Swagger en développement
- routes métiers nombreuses
- serving du frontend en production
- manifest dynamique
- logique Cloud Run

### Data Layer

Le schéma Prisma constitue un actif clé du projet.
Il porte déjà une large partie du domaine social, spatial, métier et relation client.

## 13. Forces majeures

1. Vision produit différenciante
2. Pont rare entre social, spatial et business
3. Base full-stack moderne
4. Fondations déjà profondes côté données et backend
5. Potentiel d'effet réseau
6. Forte proposition de valeur pour les sociétés une fois la traction utilisateur en place

---

## 14. Risques et points d'attention

1. Le produit peut être perçu comme trop large si le discours n'est pas hiérarchisé
2. Le backend actuel est puissant mais encore centralisé
3. Le dépôt reflète encore partiellement l'ancien récit 2Thier CRM/ERP
4. La proposition de valeur utilisateur doit rester au premier plan
5. L'univers Hive doit soutenir le produit, pas le rendre obscur
